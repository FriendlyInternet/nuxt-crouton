What changes (and what doesn't)

  • New Reprint action. Right-aligned on the UTabs tab row in OrderItems.vue, two-step confirm. Emits reprintOrder up to OrdersTab.vue.
  • Re-fires the order's OWN tickets. Resets the order's existing print_jobs from terminal status (done / failed) back to pending — no new jobs, no forced customer receipt.
  • LEDs update. After firing, the existing 2s queue poll refreshes, so the order/printer dots cycle busy → done.
  • Untouched. The per-failed-job reprint inside the Printers tab, the order lines, totals, LEDs, and every other tab-bar element stay exactly as they are.
