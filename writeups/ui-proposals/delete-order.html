What changes (and what doesn't)

  • Delete pill in the expanded row. A two-step CroutonDeleteButton in a new right-aligned footer action row, below the totals — placed below the tabs so it stays clear of the Reprint action (#1517) on the tab bar and keeps the merge small.
  • Hard delete. Removes the order and its salesOrderitems and its print_jobs (source='sales', refType='order', refId=orderId). The existing cancelled status stays as the soft alternative for real events.
  • List + POS refresh. On delete, fires the crouton:mutation hook for salesOrders so the order list and the POS client picker refresh; the deleted order stops counting toward its client's tab total.
  • Team-admin only. The DELETE endpoint requires team admin — a non-admin can't delete.
  • Untouched. The row header, LED status, tabs, item list and totals stay exactly as they are.
